## Risks & Considerations

AI in software development is not without risks. A responsible partner will be transparent about these:

### Over-Reliance on AI-Generated Code
AI-generated code is **non-deterministic** — it can produce different outputs for similar inputs, and it can introduce subtle bugs or security vulnerabilities. Every line of AI-generated code must be reviewed by a human engineer.

**Manifera's approach:** 100% human-in-the-loop. AI accelerates; humans validate.

### Intellectual Property Concerns
AI coding tools are trained on vast datasets that may include open-source code with various licenses. There are active legal debates about the copyright status of AI-generated code.

**Manifera's approach:** We maintain clear documentation of human contributions to every codebase. Our IP agreements explicitly address AI-assisted development to protect our clients' ownership rights.

### The "Speed vs. Quality" Trap
It is tempting to celebrate speed gains without verifying that quality has kept pace. AI can produce more code faster — but more code is not always better code.

**Manifera's approach:** We measure quality metrics (defect density, test coverage, code complexity scores) alongside velocity metrics. Speed without quality is just faster failure.
